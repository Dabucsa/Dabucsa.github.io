Plan: Descubre Pucón — Guía Airbnb
Sitio web estático con Next.js 16 + Tailwind CSS 4 + TypeScript, publicado en GitHub Pages desde la carpeta pucontour. Combina una guía por categorías (restaurantes, termas, aventura, naturaleza, cultura) con itinerarios sugeridos de 1 a 7 días. Se reutiliza la base de ~50 destinos de villarrica-planner/index.html expandiéndola con más contenido. Estética mobile-first (los huéspedes consultarán desde el celular), tema oscuro premium similar al villarrica-planner con glassmorphism y animaciones suaves con Framer Motion. Sin backend — todo es contenido estático con datos en archivos .ts.

Steps

Scaffold del proyecto en pucontour con Next.js 16 App Router, TypeScript, Tailwind CSS 4, Framer Motion, Lucide React — mismas dependencias y estructura que package.json y next.config.ts (output: "export", images: { unoptimized: true })

Estructura de carpetas siguiendo el patrón del workspace:

src/app/ → layout, page, globals.css, sitemap, robots

src/app/restaurantes/ → guía gastronómica

src/app/aventura/ → deportes y actividades outdoor
src/app/naturaleza/ → parques, lagos, senderos

Modelo de datos en src/data/:

places.ts — array tipado de todos los destinos con: id, name, category, description, price, duration, emoji, mapUrl (link a Google Maps), tips (consejo personal del anfitrión), bestFor (familia/aventura/pareja), weather (indoor/outdoor/both)
itineraries.ts — itinerarios pre-armados para 1, 2-3, 4-5 y 6-7 días, cada uno como array de días con array de actividades referenciando places
categories.ts — metadata de las categorías (nombre, emoji, descripción, color)

Decisions

Next.js static export sobre HTML puro: permite componentes reutilizables, tipado, y escalabilidad futura sin sacrificar la naturaleza estática para GitHub Pages
Datos en archivos .ts sobre CMS/API: simplicidad total, cero dependencias externas, fácil de editar
Tema oscuro: consistente con la estética de villarrica-planner y los otros proyectos del workspace, además luce premium
WhatsApp button: canal directo anfitrión-huésped, ya implementado en zeroaustral como referencia
No incluir sistema de afiliados/premium: este es una guía de cortesía para huéspedes, no un producto monetizable (simplifica mucho el scope)
